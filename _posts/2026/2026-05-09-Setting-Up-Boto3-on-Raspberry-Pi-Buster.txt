### apt update

Raspberry Pi OS Buster (Debian 10) reached its End of Life (EOL) in June 2024. To make `apt update` work, I had to make point the system to the new "legacy" url.

1. Update your Raspbian sources
Open the main sources file. `sudo vi /etc/apt/sources.list`
Find the line starting with deb http://raspberrypi.org buster...
Change it to: `deb https://legacy.raspbian.org/raspbian/ buster main contrib non-free rpi`
Save the change

2. Update your Raspberry Pi specific sources <== Didn't need as it's already like that
Open the Raspberry Pi list: `sudo vi /etc/apt/sources.list.d/raspi.list`
Ensure the line looks like this: deb http://archive.raspberrypi.org/debian/ buster main

```shell
sudo apt clean
sudo apt update --allow-releaseinfo-change
```

```
curl https://pyenv.run | bash
```

Add blow to `~/.bashrc` (in my Pi system, there's no `~/.bash_profiler`)

```
# Load pyenv automatically by appending
export PYENV_ROOT="$HOME/.pyenv"
[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init - bash)"
# Load pyenv-virtualenv automatically
eval "$(pyenv virtualenv-init -)"
```
Then, restart your shell.
